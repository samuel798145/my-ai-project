# 前端开发负责人 — 职责与工作安排

## 一、核心职责

### 技术架构与规范
- 制定并维护前端技术选型、架构设计方案
- 建立并推行代码规范（ESLint / Prettier / Git Commit Message）
- 主导组件库、Design System 的搭建与演进
- 评估并引入新技术，输出技术调研报告

### 开发与交付
- 负责核心模块、复杂功能的前端实现
- 把控前端整体进度，协调资源，确保按时交付
- 与后端、设计、产品协作，推动需求落地
- 对接 Figma 设计稿，保证还原度和交互一致性

### 质量保障
- 主导 Code Review，提升团队代码质量
- 制定并落地前端测试策略（单元测试 / E2E 测试）
- 使用 Lighthouse 等工具定期做性能审查，推动优化
- 接入 Sentry 等监控工具，建立错误告警机制

### 团队管理
- 带领前端团队成员，做技术指导和成长规划
- 分解任务，合理分配开发工作
- 主持前端例会，同步进展，解决阻塞
- 建立知识沉淀机制（技术文档、Notion Wiki、Storybook 组件文档）

### 流程与工程化
- 维护 CI/CD 流水线，保障构建、部署稳定
- 管理 npm 依赖，定期进行安全审计（npm audit）
- 维护 Changelog，做好版本发布管理
- 推动国际化（i18n）方案落地

- [ ] 参加周会，同步本周目标与资源

- [ ] 专注核心功能开发 / 技术攻关

- [ ] 主导 Code Review，覆盖本周主要 PR
- [ ] 与后端联调接口，解决跨端问题

- [ ] 更新 Notion 技术文档 / 规范

### 周四 — 工程化与优化
- [ ] 运行 Lighthouse 性能检测，记录指标变化
- [ ] 检查 npm 依赖安全（npm audit），处理高危漏洞

### 周五 — 复盘与沉淀
- [ ] 生成本周 Changelog，整理版本发布说明
- [ ] 参加 Sprint 回顾会，复盘本周交付与问题
