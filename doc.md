# 多 Agent 协作方案

## 一、角色定义

| 角色 | Agent 职责 | 与前端的交互点 |
|------|-----------|-------------|
| 产品 Agent | PRD 生成、需求拆解、验收标准定义 | 输出结构化需求 → 前端读取并评估工作量、识别技术风险 |
| UI Agent | 解析 Figma 设计稿、输出组件规范与标注 | 前端消费 Design Token、完成 UI 还原并验证 |
| 后端 Agent | API 文档生成、Mock 数据维护、接口变更通知 | 前端依赖接口契约进行联调，消费 OpenAPI Spec |
| 前端 Agent | 功能实现、Code Review、性能优化、文档维护 | 汇总上游输出，交付可运行、可测试的代码 |

## 二、协作流程（Sprint 维度）

```
┌─────────────────────────────────────────────────┐
│                   Sprint 启动                    │

         │  评审需求         │
         │  评估工作量/风险  │
         └──┬──────────┬───┘
            │          │
   ┌────────▼──┐   ┌───▼────────┐
   │  UI Agent │   │  后端 Agent │
   │ 设计稿输出 │   │ 接口文档    │
   │ 组件规范  │   │ Mock 数据   │
   └────────┬──┘   └───┬────────┘
            │          │
         ┌──▼──────────▼───┐
         │    前端 Agent    │
         │  并行开发         │
         │  对接设计 + 接口  │
         └────────┬────────┘
                  │
         ┌────────▼────────┐
         │  Code Review    │
         │  → CI/CD        │
         │  → 上线          │
         └────────┬────────┘
                  │
         ┌────────▼────────┐
         │  各 Agent 同步   │
         │  Changelog / 文档│
         └─────────────────┘
```

## 三、通信协议

### 信息格式
- Agent 间统一使用 **结构化 Markdown** 或 **JSON** 传递信息
- 需求文档：PRD 使用 Markdown，包含验收标准（AC）
- 接口契约：使用 **OpenAPI 3.0 Spec**，锁定字段类型与枚举值
- 设计交付：Figma 链接 + 导出的 Design Token JSON

### 通知机制
- 变更通知：通过 **Slack / Lark** 推送到对应频道
  - `#frontend` — 前端内部
  - `#api-changes` — 后端接口变更
  - `#design-handoff` — 设计稿更新
  - `#ci-alerts` — CI/CD 状态
- 阻塞问题：立即同步，不等下次例会

### 协作工具矩阵

| 工具 | 用途 |
|------|------|
| Jira | Sprint 管理、任务跟踪 |
| Figma | 设计稿交付、标注 |
| Notion | 技术文档、规范 Wiki |
| GitHub | 代码托管、PR、Code Review |
| Storybook | 组件文档 |
| Sentry | 前端错误监控 |
| Slack / Lark | 即时通知、跨团队沟通 |

## 四、各阶段交付物

### 产品 Agent → 前端 Agent
```markdown
## 需求名称
## 背景与目标
## 用户故事
## 验收标准（AC）
## 优先级 & 截止日期
## 设计稿链接
```

### UI Agent → 前端 Agent
```markdown
## 组件清单
## Figma 链接
## Design Token
## 交互说明
## 边界 & 异常态
```

### 后端 Agent → 前端 Agent
```markdown
## 接口列表（OpenAPI Spec）
## Mock 地址
## 字段说明
## 变更记录
```

### 前端 Agent → 全体
```markdown
## 本周完成
## 下周计划
## 风险 & 阻塞
## Changelog
```

## 五、质量卡点

| 卡点 | 负责 Agent | 标准 |
|------|-----------|------|
| 需求评审 | 前端 + 产品 | 无歧义、有 AC |
| 设计走查 | 前端 + UI | 还原度 ≥ 95% |
| 接口联调 | 前端 + 后端 | 0 字段不一致 |
| Code Review | 前端 Agent | 0 Error，Coverage ≥ 70% |
| 性能检测 | 前端 Agent | Lighthouse ≥ 90 |
| 上线前验收 | 产品 Agent | AC 全部通过 |

---

> 本文档为前端负责人主导的多 Agent 协作规范，各角色可根据实际项目情况迭代更新。
